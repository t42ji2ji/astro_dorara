# 資源搜尋指南

用 web search 豐富講義內容。目標不是產出「延伸閱讀清單」，而是**找出高品質內容，提取觀點，直接融入講義**。

---

## 什麼時候搜尋

- 需要專家觀點增加可信度
- 需要案例或數據支撐論點
- 想確認最佳實踐或常見錯誤
- 覺得內容太淺，需要深度

---

## 搜尋策略

### 找高評價資源

```
書籍：
"{主題}" best book recommendation
"{主題}" book review Goodreads
"{主題}" "must read"

文章：
"{主題}" definitive guide
"{主題}" comprehensive overview

專家：
"{主題}" expert interview
"{主題}" creator founder  （找原創者）
```

### 驗證品質

```
"{書名}" review
"{作者}" credentials
"{概念}" research evidence  （找學術支撐）
"{概念}" criticism  （找批評，了解弱點）
```

### 按課程類型搜尋

**工具類**（Claude Code、Cursor）：
```
"{工具}" official documentation
"{工具}" tips and tricks
"{工具}" vs {競品} comparison
```

**方法論類**（超速學習、原子習慣）：
```
"{方法論}" author interview
"{方法論}" case study implementation
"{方法論}" research evidence
```

**技能類**（Prompt Engineering、寫作）：
```
"{技能}" common mistakes
"{技能}" before after comparison
"{技能}" expert to follow
```

---

## 品質指標

好資源的特徵：
- ✅ Goodreads 4.0+ 或 Amazon 4.5+
- ✅ 被多個獨立來源推薦
- ✅ 作者是該領域公認專家
- ✅ 有實際範例或數據支撐

避免：
- ❌ 內容農場、SEO 灌水文
- ❌ 純書摘（沒有自己見解）
- ❌ 過時資訊（看發布日期）
- ❌ 純理論沒有範例

---

## 優先順序

1. **原創者 > 二手詮釋**
   - 找 Cathy Moore 本人的說法，不是別人介紹她

2. **專書 > 文章**
   - 書籍通常有更完整的論述

3. **有數據 > 純觀點**
   - 優先找有研究支撐的內容

4. **實務 > 理論**
   - 優先找有實際案例的內容

## 內容提取模板

對於找到的資源，提取以下內容：

```markdown
## 資源：{資源名稱}

**來源**：{作者/出處}
**為什麼重要**：{一句話說明這個資源的價值}

### 核心主張
- ...

### 可引用的金句
> "..."

### 框架/模型
- ...

### 可融入講義的觀點

| 觀點 | 適合放在哪 | 如何融入 |
|-----|-----------|---------|
| ... | ... | ... |
```

---

## 融入方式

找到資源後，不只是列出來，還要建議**怎麼用**：

| 融入方式 | 說明 | 範例 |
|---------|------|------|
| **開場破題** | 用專家觀點點出問題重要性 | 「Cathy Moore 說：『80% 的培訓內容是浪費的』」 |
| **理論支撐** | 為你的做法提供權威背書 | 「這個原則來自 Gagné 的研究」 |
| **金句引用** | 用精準表達替代白話說明 | 直接引用專家的精闘表達 |
| **案例補充** | 用專家的案例豐富說明 | 「Google 的做法是...」 |
| **反面教材** | 用專家指出的錯誤強化對比 | 「常見錯誤是...」 |
| **延伸思考** | 在章節結尾引導深入 | 「想深入了解，可以看...」 |

---

## 輸出格式

當建議融入講義時，使用這個格式：

```markdown
## 建議融入的內容

### 針對 {章節名稱}

**現有內容**：
> {引用講義原文}

**建議補充**：
{具體的補充內容，已經寫好可以直接貼}

**來源**：{資源名稱}，{作者}
```

---

## 執行原則

### 品質 > 數量
- 寧可深入挖掘 2-3 個高品質資源
- 每個觀點都要能說出「為什麼值得放進講義」

### 融入 > 附加
- 目標是讓講義本身更豐富，不是產出附錄
- 觀點要自然融入講義脈絡，不是硬塞引用

### 驗證 > 信任
- 交叉驗證重要資訊
- 注意發布日期（AI 領域尤其重要）
- 找批評意見，了解資源的局限
