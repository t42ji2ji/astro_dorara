# 教學方法論選擇指南

選對教學方法比努力更重要。這份指南幫你根據課程目標選擇合適的方法。

---

## 快速選擇

```
你的課程目標是什麼？
│
├─ 改變特定行為 → Action Mapping
│   （銷售話術、客服應對、工具使用）
│
├─ 培養複雜專業技能 → 4C/ID
│   （程式設計、醫療診斷、多技能整合）
│
├─ 教會標準流程 → Gagné 九大事件
│   （設備操作、安全程序、合規培訓）
│
└─ 建立深層理解 → Merrill + UbD
    （領導力、策略思考、可遷移的能力）
```

**大多數情況**：你的課程是**行為改變**類型，用 **Action Mapping** 就對了。

---

## Action Mapping

### 什麼時候用

- 學員要**做**不一樣的事
- 目標是行為改變，不是知識累積
- 可以讓學員在安全環境中試錯

### 核心流程

1. **定義目標行為**：「學完能做什麼？」（必須可觀察）
2. **拆解子行為**：大行為拆成小行為
3. **先設計練習**：不是先寫知識點，是先設計練習
4. **最少資訊**：只給完成練習需要的最少知識

### 練習設計原則

好的練習：
- 有具體情境
- 讓學員「產出」而不是「選擇」
- 反饋顯示「後果」而不只是「對錯」
- 和真實使用場景相似

壞的練習：
- 選擇題（選對不代表會做）
- 沒有情境的抽象問題
- 只說對錯，不說後果

### 範例

**課程目標**：寫出讓 AI 正確完成任務的 prompt

**子行為**：
1. 判斷任務適不適合交給 AI
2. 把模糊需求轉成明確指令
3. 提供足夠的上下文
4. 指定輸出格式
5. 發現 AI 誤解時，調整 prompt 修正

每個子行為 = 一個單元 = 一組練習

## Gagné 九大事件

### 什麼時候用

- 高風險操作，錯誤代價太高
- 流程必須正確，不能靠試錯
- 合規培訓、安全程序、設備操作

### 九個步驟

| 步驟 | 目的 | 做法 |
|-----|------|------|
| 1. 引起注意 | 激活學員 | 震撼數據、問題、故事 |
| 2. 告知目標 | 設定期望 | 「學完能做什麼」 |
| 3. 喚起舊知 | 連結已知 | 「你應該已經會 X」 |
| 4. 呈現教材 | 展示內容 | 正例與反例 |
| 5. 提供輔導 | 認知鷹架 | 每一步都有指導 |
| 6. 引發表現 | 讓學員做 | 練習操作 |
| 7. 提供回饋 | 立即回應 | 具體說明對錯 |
| 8. 評估表現 | 確認達標 | 測驗或檢核 |
| 9. 促進遷移 | 連結真實 | 「回到工作時這樣用」 |

### 與 Action Mapping 的差異

- Action Mapping：讓學員先試錯再學習
- Gagné：確保學員不會建立錯誤的心理模型

**風險高 → 用 Gagné；風險低 → 用 Action Mapping**

## 4C/ID（四元素教學設計）

### 什麼時候用

- 培養專家級的複雜技能
- 多種子技能要同時運作
- 長期養成計畫（Bootcamp、專業培訓）

### 四個元素

1. **學習任務（Learning Tasks）**
   - 從一開始就給完整的真實任務
   - 任務按難度分級，從簡單到複雜
   - 提供鷹架支持，逐漸撤除

2. **支持性資訊（Supportive Information）**
   - 對應「非重複性技能」（推理、問題解決）
   - 在任務開始前提供
   - 幫助學員建立心理模型

3. **程序性資訊（Procedural Information）**
   - 對應「重複性技能」（標準操作）
   - Just-in-time：做的時候才給
   - 像工具提示、操作手冊

4. **部分任務練習（Part-task Practice）**
   - 針對必須自動化的子技能
   - 重複練習直到熟練
   - 例如：打字速度、急救按壓頻率

### 與 Action Mapping 的差異

- Action Mapping：把大目標拆成幾個行為
- 4C/ID：從一開始就讓學員接觸完整任務，逐步提高難度

## Merrill 首要教學原理

### 什麼時候用

- 需要認知激活與深度理解
- 學員要能把技能遷移到新情境
- 問題導向的學習

### 五個原理

1. **問題導向**：學習圍繞真實問題展開
2. **激活舊知**：把新知識錨定在已知的東西上
3. **展示（Show Me）**：示範怎麼做，包含正例與反例
4. **應用（Let Me）**：讓學員練習，提供逐漸撤除的指導
5. **整合**：讓學員把新技能融入日常（反思、公開展示）

### 與 Action Mapping 的差異

- Merrill 更強調「展示」階段
- 適合需要先看示範才能模仿的技能

## 常見組合

| 課程類型 | 範例 | 建議組合 |
|---------|-----|---------|
| 工具入門 | Claude Code、Cursor、Excel | Action Mapping + Gagné（操作步驟的鷹架） |
| 方法論導讀 | 超速學習、原子習慣 | Action Mapping + UbD（確保理解能遷移） |
| 技能培養 | Prompt Engineering、寫作 | Merrill（問題導向）+ Action Mapping（大量練習） |
| 專業養成 | 全端工程師、產品經理 | 4C/ID（完整任務）+ Gagné（程序性知識） |

## 選擇決策表

| 問題 | 是 → | 否 → |
|-----|------|------|
| 錯誤代價高嗎？ | Gagné | 繼續問 |
| 需要多技能整合嗎？ | 4C/ID | 繼續問 |
| 需要深度理解和遷移嗎？ | Merrill | Action Mapping |

**不確定時，就用 Action Mapping**。它適用於大多數課程。
