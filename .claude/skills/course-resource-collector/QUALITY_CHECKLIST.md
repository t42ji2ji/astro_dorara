# 資源品質檢核清單

在將資源加入清單前，確認通過以下檢核。

---

## 必要條件（全部要通過）

### 相關性
- [ ] 直接與課程主題相關（不是「相關領域」）
- [ ] 對目標學員有實際幫助
- [ ] 補充講義中沒有涵蓋的面向

### 可信度
- [ ] 來源可辨識（知道是誰寫的/做的）
- [ ] 作者有相關背景或作品可查證
- [ ] 不是純行銷內容或業配文

### 可訪問性
- [ ] 連結可正常開啟
- [ ] 核心內容可免費取得（付費內容需標註）
- [ ] 不需要特殊軟體或權限

---

## 分類檢核

### 📚 書籍

| 檢核項目 | 說明 |
|---------|------|
| [ ] 評價良好 | Goodreads 4.0+ 或豆瓣 8.0+ |
| [ ] 有中文版 | 如果學員主要用中文，優先找有翻譯的 |
| [ ] 不是過時 | 技術書 3 年內，方法論書可放寬 |
| [ ] 有試讀 | 最好有試讀章節可以先看 |

### 📝 文章

| 檢核項目 | 說明 |
|---------|------|
| [ ] 發布日期 | 技術文章 2 年內，概念文章 5 年內 |
| [ ] 有實例 | 不是純理論，有程式碼/截圖/案例 |
| [ ] 原創內容 | 不是轉載或翻譯（找原始來源） |
| [ ] 深度足夠 | 不是表面介紹，有深入分析 |

### 🔧 工具

| 檢核項目 | 說明 |
|---------|------|
| [ ] 仍在維護 | 最近 6 個月有更新 |
| [ ] 有文件 | 有基本的使用說明 |
| [ ] 免費可用 | 至少有免費版或試用期 |
| [ ] 穩定可靠 | 不是剛出的 beta 版（除非特別標註） |

### 🎥 影片課程

| 檢核項目 | 說明 |
|---------|------|
| [ ] 影音品質 | 畫質清晰、收音清楚 |
| [ ] 有字幕 | 最好有英文或中文字幕 |
| [ ] 評價良好 | 平台評分 4.0+ |
| [ ] 時長合理 | 標註總時長，讓學員評估 |

### 👤 專家追蹤

| 檢核項目 | 說明 |
|---------|------|
| [ ] 仍在活躍 | 最近 3 個月有發文 |
| [ ] 有代表作 | 有知名文章、專案或演講 |
| [ ] 內容實用 | 不只是轉發或評論 |
| [ ] 互動良好 | 會回覆問題或參與討論 |

## 紅旗警告（看到這些要小心）

### 來源紅旗
- ❌ 網站充滿廣告，難以閱讀
- ❌ 來源不明，沒有作者資訊
- ❌ 標題黨（「10 個你不知道的 XXX」「震驚！」）
- ❌ 內容和標題不符
- ❌ 複製貼上自其他網站

### 內容紅旗
- ❌ 只有概念沒有實例
- ❌ 錯誤資訊（與官方文件矛盾）
- ❌ 過度簡化（導致誤解）
- ❌ 過度推銷特定產品
- ❌ 明顯的 AI 生成內容（沒有個人觀點）

### 時效紅旗
- ❌ 技術文章超過 2 年沒更新
- ❌ 提到的版本已過時
- ❌ 連結到已下線的服務
- ❌ 截圖/介面與現在不符

---

## 加分項目（有的話更好）

- ⭐ 有中文版本或中文社群
- ⭐ 作者有持續更新
- ⭐ 有配套的練習或範本
- ⭐ 有活躍的討論區或社群
- ⭐ 提供原始碼或可下載資源
- ⭐ 有不同難度的版本

---

## 驗證步驟

蒐集完資源後，執行以下驗證：

1. **連結測試**
   - 用 WebFetch 確認每個連結可訪問
   - 記錄任何需要登入或付費的資源

2. **交叉驗證**
   - 對於重要資訊，找第二個來源確認
   - 對於工具推薦，確認不只一個地方推薦

3. **時效檢查**
   - 確認發布/更新日期
   - 技術類超過 2 年的要特別標註

4. **偏見檢查**
   - 確認不是只推薦某一派的觀點
   - 如果有爭議，呈現不同立場

---

## 常見錯誤

| 錯誤 | 問題 | 修正 |
|-----|------|------|
| 只找英文資源 | 中文學員難以使用 | 補充繁體中文資源 |
| 只找免費資源 | 可能錯過高品質付費內容 | 標註付費資源，說明值不值得 |
| 太多資源 | 學員不知道從哪開始 | 標註「必讀」，限制每類 3-5 個 |
| 太少資源 | 涵蓋面不足 | 至少 3 種類型，每類 2+ 個 |
| 只有基礎資源 | 進階學員沒東西看 | 分難度層級推薦 |
