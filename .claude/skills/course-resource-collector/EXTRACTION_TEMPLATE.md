# 內容提取模板

從高品質資源中提取可融入講義的內容。

---

## 資源基本資訊

```markdown
## 資源：{名稱}

**類型**：書籍 / 文章 / 訪談 / 課程
**作者**：{姓名}，{背景/為什麼可信}
**來源**：{連結}
**評價指標**：{Goodreads 分數 / 引用次數 / 推薦來源}
```

---

## 提取清單

### 1. 核心主張

作者最重要的 1-3 個觀點。用一句話表達。

```markdown
### 核心主張

1. {主張一}
2. {主張二}
3. {主張三}
```

**判斷標準**：
- 這個觀點是否貫穿整個資源？
- 這個觀點是否和其他人的說法不同？
- 這個觀點是否有足夠的證據支撐？

---

### 2. 金句

可以直接引用的精準表達。好的金句特徵：
- 用簡單的語言說出深刻的道理
- 有記憶點，容易被轉述
- 能引發讀者的「啊哈」時刻

```markdown
### 可引用的金句

> "{金句一}"
> — {作者}，{出處}

> "{金句二}"
```

**使用場景**：
- 章節開頭破題
- 強調重點時引用
- 總結時呼應

---

### 3. 框架/模型

作者提出的結構化思考方式。

```markdown
### 框架/模型

**{框架名稱}**
{簡短描述}

步驟/元素：
1. ...
2. ...
3. ...
```

**價值判斷**：
- 這個框架是否比現有的說法更清晰？
- 這個框架是否有實際應用的案例？
- 這個框架是否容易記住和使用？

---

### 4. 案例/數據

支撐觀點的具體證據。

```markdown
### 案例/數據

**案例：{案例名稱}**
- 情境：{背景}
- 做法：{怎麼做}
- 結果：{成效}
- 教訓：{可以學到什麼}

**數據**：
- {統計數字}（來源：{研究/報告名稱}）
```

**品質標準**：
- 案例是否具體、可驗證？
- 數據是否來自可信來源？
- 是否有對照組或前後對比？

---

### 5. 常見錯誤/反模式

作者指出的「不要這樣做」。

```markdown
### 常見錯誤

| 錯誤做法 | 為什麼是錯的 | 正確做法 |
|---------|------------|---------|
| {錯誤一} | {原因} | {替代方案} |
| {錯誤二} | {原因} | {替代方案} |
```

**價值**：
- 幫助讀者避免踩坑
- 強化正確做法的理由
- 讓內容更有實戰感

---

### 6. 獨特視角

這個資源跟其他資源不同的地方。

```markdown
### 獨特視角

**主流觀點**：{大多數人怎麼說}

**這個資源的不同**：{作者的獨特觀點}

**為什麼值得採納**：{理由}
```

---

## 融入建議

提取完內容後，具體建議怎麼用：

```markdown
## 融入建議

### 建議 1：{章節名稱}

**現有內容**：
> {引用講義原文}

**建議補充**：
{具體的補充文字，可以直接複製貼上}

**融入方式**：開場破題 / 理論支撐 / 金句引用 / 案例補充 / 反面教材

**來源**：{資源名稱}，{作者}

---

### 建議 2：{章節名稱}
...
```

---

## 提取品質檢核

完成提取後，確認：

- [ ] 每個觀點都能回答「為什麼這個值得放進講義？」
- [ ] 金句是直接引用，不是改寫
- [ ] 案例和數據有標註來源
- [ ] 融入建議具體到可以直接執行
- [ ] 沒有過度依賴單一資源（至少 2 個來源交叉驗證）
