## 這個單元你會學會做什麼

把你原本準備的知識內容，分類成**練習前必須知道**、**查表就好**、**選讀**三類，然後刪掉其他的。

🟡 進階 ｜ 約 20 分鐘

---

## 快速說明

設計完練習之後，回頭看你原本準備的教學內容，你會發現一個問題：

**太多了。**

大多數課程塞太多知識，原因是設計者覺得**這個學員應該知道**、**那個也很重要**、**這個背景很有趣**。

Cathy Moore 說得很直接：

> 資訊是我們唯一擁有的東西，所以我們對它產生了不健康的執著。

但學員不是來聽你演講的，他們是來學會做某件事的。資訊過載不只浪費時間，還會**降低學習效果**。認知科學研究顯示，當工作記憶被過多資訊塞滿，學員反而什麼都記不住。

Action Mapping 的原則是：**只給完成練習需要的最少資訊。**

### 知識內容的四種命運

| 類型 | 處理方式 | 放哪裡 |
|-----|---------|-------|
| 完成練習必須知道的 | 保留，放練習前 | **快速說明**段落 |
| 做的時候可以查的 | 保留，獨立出來 | **速查表**或**附錄** |
| 有興趣再看的背景 | 保留，標明選讀 | **為什麼**段落 |
| 以上都不是 | 刪掉 | 垃圾桶 |

### 怎麼判斷？

問自己一個問題：

> 如果學員不知道這個，他能完成練習嗎？

- 能 → 不用放在練習前
- 不能 → 放在練習前，簡短說明

再問：

> 學員做練習的時候，會需要查這個嗎？

- 會 → 做成速查表
- 不會 → 考慮刪掉或放選讀

## 練習

### 情境

你正在設計 **Git 入門**課程。
目標行為是：**用 Git 建立分支、切換分支、合併分支**。

你準備了以下知識內容：

1. Git 的歷史（Linus Torvalds 在 2005 年創建）
2. 版本控制的概念（什麼是版本控制、為什麼需要）
3. Git vs SVN 的比較
4. 分支的概念（什麼是分支、為什麼要用分支）
5. git branch 指令的用法
6. git checkout 指令的用法
7. git merge 指令的用法
8. 合併衝突的處理方式
9. Git 的內部運作原理（blob、tree、commit 物件）
10. 常用 Git 指令速查表

### 任務

把這 10 項內容分類：

| 類型 | 內容編號 |
|-----|---------|
| 練習前必須知道 | |
| 速查表 | |
| 選讀 | |
| 刪掉 | |

**你的分類：**

```
練習前必須知道：


速查表：


選讀：


刪掉：

```

---

### 參考答案

```
練習前必須知道：4, 5, 6, 7
（分支概念 + 三個指令的基本用法。沒這些，練習做不了。）

速查表：10
（做練習時忘記指令可以查。）

選讀：2, 8
（版本控制概念可以幫助理解，但不知道也能做練習。
 合併衝突是進階內容，這堂課的練習可能還不會遇到。）

刪掉：1, 3, 9
（Git 歷史是有趣的冷知識，但對學會用 Git 沒幫助。
 Git vs SVN 對初學者沒意義，他們根本沒用過 SVN。
 內部原理太深，初學者不需要知道。）
```

**注意**：這個分類會因為你的目標行為不同而改變。如果目標是**理解 Git 的設計哲學**，那內部原理就該放練習前了。

## 砍內容的心理障礙

你可能會想：**可是這些內容我都花時間準備了...**

幾個思考：

1. **學員的時間比你的時間貴。** 你花 2 小時寫的內容，如果 100 個學員各浪費 10 分鐘讀，那是 1000 分鐘的浪費。

2. **資訊過載會降低學習效果。** 認知負荷理論告訴我們：工作記憶容量有限，塞太多東西，學員反而什麼都記不住。Julie Dirksen 在《Design for How People Learn》中用一個比喻：想像學員的腦袋是一個小房間，你不斷往裡面塞傢俱，最後他連走路的空間都沒有。

3. **砍掉不是消失。** 那些內容可以變成：
   - 部落格文章
   - 進階課程的內容
   - **延伸閱讀**的連結
   - **做的時候查**的速查表

4. **「拉取」比「推送」有效。** 與其事先塞給學員一堆資訊，不如讓他們在需要的時候自己去查。這是 Action Mapping 的核心觀念之一：資訊應該被學員主動拉取，而不是被動接收。

## 檢查表

精簡完內容後，用這個清單檢查：

- [ ] 練習前的說明不超過 5 個重點
- [ ] 每個重點都能回答**這跟完成練習有什麼關係？**
- [ ] 可以查表的東西獨立成速查表
- [ ] 背景知識標明**選讀**，放在練習之後
- [ ] 刪掉了至少 30% 原本準備的內容（認真的）

## 這堂課的結尾

你現在會了：

1. **定義目標行為**：把**學會 X** 轉成可觀察的行為
2. **設計練習活動**：用情境和任務讓學員真的**做**
3. **精簡知識內容**：只留完成練習需要的最少資訊

這三步是 Action Mapping 的核心。其他的步驟（原型測試、批次製作等）是執行細節，知道這三步就能開始設計有效的課程了。

### 最後的練習

回到最開始：你準備教的那個主題。

現在，用這三步設計它的第一個單元：

1. 這個單元要讓學員**做到**什麼？（一句話）
2. 設計一個練習活動（情境 + 任務 + 參考答案）
3. 列出完成這個練習需要的最少知識（不超過 5 點）

做完這個練習，你就真的會 Action Mapping 了。
